## Steps
1. **Recon (read-only).** Use `opencli-browser` to launch Playwright with a
   persistent user-data-dir, complete SSO+2FA once manually, then capture for
   `projectId=2548`:
   - route map (list / detail / create / update / delete views)
   - list XHR (filters, pagination, sort)
   - single-item detail XHR
   - create/update/delete XHR (including CSRF header pattern, payload schema)
   - any project-scoped metadata XHR (field schema, enums, dictionaries)
   Write findings to `docs/recon/mpline.md`. **Never mutate during recon.**
2. **Adapter scaffold.** Generate `adapters/atlas/` per
   `opencli-adapter-author` conventions: selector-first targets, agent-native
   envelope, `--json` output, `--dry-run` default for any mutating command.
   Every command takes `--project-id` (default from env `BANMA_PROJECT_ID`).
3. **Commands (in this order — ship incrementally, verify each):**
   - `auth login` — Playwright headed; persists `storageState` (encrypted via macOS Keychain).
   - `list` — items in project; filters: `status`, `assignee`, `since/until`, `q`; table or `--json`.
   - `show <itemId>` — full item payload + linked metadata.
   - `export` — bulk fetch → CSV / JSON / Parquet; supports `--since` for incremental.
   - `fill` — for each item, render a user prompt template (Jinja-like) with
     item fields, call a configured LLM (env-driven, default off), produce
     field updates, stage to file. Real submit only with `--apply`.
   - `import <file>` — CSV/XLSX → zod schema validation → posts create/update.
     Always dry-run first; `--apply` to commit.
4. **Verify loop.** For each command: `opencli verify`, capture trace
   artifacts under `.opencli/trace/`, exit non-zero on schema or selector
   drift. Wire `opencli-autofix`.
5. **Skill packaging.** Emit `skills/atlas/SKILL.md` so Claude Code
   agents can invoke commands without reading source.
